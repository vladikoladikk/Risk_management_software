# **Программное средство управления рисками в проектной деятельности**

### **Краткое описание проекта**

Разрабатываемое программное средство представляет собой микросервисную систему управления рисками в проектной деятельности, предназначенную для поддержки полного жизненного цикла рисков: от идентификации до контроля и отчётности. Система обеспечивает интеграцию функциональных модулей для ведения проектов, идентификации и оценки рисков, планирования мероприятий по реагированию и формирования аналитической отчётности. В основе архитектуры лежит трёхслойная структура каждого сервиса и событийно-ориентированное взаимодействие, что позволяет достичь модульности, гибкости и масштабируемости решения.

### **Цель проекта**

Основной целью проекта является повышение эффективности управления рисками в проектной деятельности за счёт внедрения автоматизированной системы поддержки принятия решений на основе микросервисной архитектуры.

### **Основные возможности проекта**

*   Управление проектами и их этапами, включая интеграцию с корпоративными системами управления проектами
*   Настройка методологии управления рисками: матриц вероятности и влияния, категорий рисков и порогов реагирования
*   Формирование реестра заинтересованных сторон (стейкхолдеров) проекта
*   Идентификация рисков: создание карточек рисков, ведение общего реестра и импорт данных
*   Качественный и количественный анализ рисков с автоматическим расчётом уровня риска и построением матрицы вероятности и влияния
*   Планирование реагирования на риски: выбор стратегий и разработка плана мероприятий с интеграцией в задачи проекта
*   Мониторинг и контроль рисков: отслеживание статусов рисков и мероприятий, автоматические уведомления и ведение аудиторского следа
*   Формирование настраиваемых дашбордов и отчетности (PDF, Excel) для анализа и коммуникации со стейкхолдерами
*   Ведение базы знаний (Lessons Learned) по рискам завершенных проектов

## **Архитектура**

### C4-модель

* С4_container.png 
Данная диаграмма C4 на контейнерном уровне описывает архитектуру системы управления рисками. 

**Основные компоненты:**

- **Клиентское приложение** - веб-интерфейс для пользователей
- **API Gateway** - единая точка входа для маршрутизации запросов
- **Микросервисы**: Управления рисками, Проектов, Уведомлений
- **Брокер сообщений** - для асинхронного взаимодействия между сервисами
- **База данных** - хранение данных всех сервисов
- **Система аутентификации** - проверка подлинности пользователей
- **Почтовый сервер** - отправка уведомлений

**Пользователи**: Менеджер проекта и Участник проекта.

Система построена по микросервисной архитектуре с разделением ответственности: отдельные сервисы отвечают за управление рисками, проектами и уведомлениями, что обеспечивает модульность и масштабируемость решения.
